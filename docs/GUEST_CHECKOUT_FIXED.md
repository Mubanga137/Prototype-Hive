```typescript

  .insert(insertPayload)
  .select("id, tracking_token")  // ✅ Now captures token

```

**Save token to localStorage & redirect for guests:**
```typescript

  setTimeout(() => {
    navigate(`/track-order/${orderId}?token=${trackingToken}`, { replace: true });
```

## What This Achieves

✅ **Guests can checkout WITHOUT logging in**
- No auth gate modal
- Direct form submission
- Frictionless P2P checkout

✅ **Tracking token auto-generated by Supabase**
- `tracking_token` UUID on order insert
- Returned in `.select()` response
- Unique, secure, immutable

✅ **Token stored in browser localStorage**
- Key: `hive_guest_orders`
- Value: `{ orderId, trackingToken, timestamp }`
- Survives page refresh
- Persists guest session permanently

✅ **Guest redirected to order tracking**
- Route: `/track-order/:order_id?token=[tracking_token]`
- Token passed as URL param
- Can restore from localStorage on page reload

✅ **RLS enforces security server-side**
- Database allows anon INSERT
- Guest SELECT only via RPC with token validation
- Token matches server record; no enumeration possible

## Guest Checkout Flow (Now Working)

```
1. Guest fills checkout form (no login required)
2. Submits → .insert() to orders table (anon role)
3. Supabase auto-generates tracking_token UUID
4. Response includes: { id, tracking_token }
5. Save to localStorage: { orderId, trackingToken }
6. Redirect to: /track-order/:id?token=[token]
7. Guest views order live
8. localStorage persists session (no re-login)
9. Refresh page → localStorage restores session
```

---

## Authenticated User Flow (Unchanged)

```
1. Authenticated user fills checkout form
2. Submits → .insert() to orders table (authenticated role)
3. buyer_id = auth.uid() (captured in payload)
4. Redirect to /track-orders (multi-order view)
5. Query fetches all orders where buyer_id = auth.uid()
6. No token needed (RLS checks buyer_id)
```
