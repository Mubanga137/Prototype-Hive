## Commit 1: Core Orchestration Layer

commit: Implement complete messaging orchestration layer in checkout flows

This implements the critical missing piece: the orchestration layer that
triggers ALL downstream messaging systems after successful order creation.

BREAKING CHANGE: Before this, orders were created but NO notifications
were sent to guests, customers, vendors, or riders.

Changes:
- CheckoutDrawer.tsx: Add parallel orchestration with Promise.allSettled
- CartDrawer.tsx: Add parallel orchestration for multi-item orders
- Both use try/catch in async IIFE to properly handle errors
- All messaging runs non-blocking to prevent UI delays
- Comprehensive logging at every stage

The orchestration follows this pattern:
1. Order created ✅
2. Send customer receipt (parallel)
3. Send vendor notification (parallel)
4. Trigger external webhook (parallel)
5. All tasks complete (success count logged)
6. UI updates with success state

Results:
- Guests: Order confirmation receipt
- Customers: Order confirmation in message inbox
- Vendors: New order notification with details
- Riders: Delivery assignment notification
- All happening in parallel without blocking the UI

Fixes: #ISSUE_NUMBER (if applicable)
