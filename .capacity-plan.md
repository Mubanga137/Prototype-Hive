Implement a capacity-based system where vendors and gig workers have a limited `order_capacity` (defaults to 50 at registration). This creates a psychological/economic pressure to drive premium recharges.

## TASK 4: THE INVISIBLE TOLL DEDUCTION (Supabase Hook)

### When to Deduct
On successful order completion (when order status moves to "delivered" or payment is confirmed):
- Deduct **1** from vendor's `order_capacity`
- Only execute after order is fully confirmed

### Implementation
1. In `CheckoutDrawer.tsx` (after OTP generation succeeds):
   - Call a new function `deductVendorCapacity(sme_id)`
2. In `CartDrawer.tsx` (after all cart items inserted):
   - Call `deductVendorCapacity(store_id)` for each item's quantity? Or just once per order?
   - **Decision**: Deduct 1 per unique order, not per quantity
3. Create utility function `src/lib/capacityDeduction.ts`:
   ```ts
   export async function deductVendorCapacity(sme_id: number): Promise<void>
   ```
   - Fetch the vendor's profile via sme_stores.owner_user_id
   - Safely deduct 1 from order_capacity (use RPC or Supabase trigger)
   - Never go below 0

### Supabase-side (Backend Logic)
- **Option A** (Trigger): Create a PostgreSQL trigger on orders table ŌåÆ decrement vendor capacity
- **Option B** (RPC): Create a Supabase function `decrement_vendor_capacity(sme_id)`
- **Recommendation**: Use RPC for safety + auditability
